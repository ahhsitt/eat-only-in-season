# Feature Specification: 活泼年轻风沉浸式 UI 重设计

**Feature Branch**: `005-page-ui-redesign`
**Created**: 2026-01-13
**Status**: Draft
**Input**: User description: "再重新设计一下整体的页面UI"

## Clarifications

### Session 2026-01-13

- Q: 色彩方案具体选择什么主色调方向？ → A: 珊瑚橙粉系（温暖、年轻、食物关联）
- Q: 动画性能降级策略如何处理？ → A: 遵循系统 prefers-reduced-motion 设置自动降级

## Overview

将现有的鼠尾草绿色调 Ant Design UI 重新设计为活泼年轻的沉浸式体验。通过明亮色彩、有趣的动效、插画元素和全屏沉浸式布局，让"不时不食"应用更具亲和力和视觉吸引力，打造适合年轻用户群体的现代化应季食谱发现体验。

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 沉浸式首页搜索体验 (Priority: P1)

用户打开应用时，看到一个充满活力的全屏欢迎界面。背景采用明亮渐变色或季节性插画，城市搜索框位于视觉中心位置，整体给人清新、年轻、有趣的第一印象。搜索时有流畅的动画反馈，让用户感受到应用的活力。

**Why this priority**: 首页是用户的第一接触点，沉浸式设计能立即抓住年轻用户的注意力，建立品牌印象。

**Independent Test**: 可以独立验证首页的视觉效果和交互体验，通过打开首页、输入城市、观察动画效果等操作验证。

**Acceptance Scenarios**:

1. **Given** 用户首次打开应用, **When** 页面加载完成, **Then** 看到全屏沉浸式欢迎界面，色彩明亮活泼，有品牌特色
2. **Given** 用户在城市搜索框输入内容, **When** 进行搜索操作, **Then** 有流畅的动画过渡效果，视觉反馈有趣
3. **Given** 搜索完成后, **When** 显示食材列表, **Then** 页面以流畅动画过渡到食材展示区域
4. **Given** 用户在首页浏览, **When** 与各元素交互, **Then** 交互反馈生动有趣（如悬停效果、点击动画）

---

### User Story 2 - 活力食材选择体验 (Priority: P1)

用户在食材选择区域看到的是色彩丰富、排版活泼的食材展示。每种食材都有独特的视觉呈现（如图标、颜色标签），选择食材时有愉悦的动效反馈。整体布局打破传统的规整网格，呈现年轻化的视觉效果。

**Why this priority**: 食材选择是核心交互流程，活泼的视觉设计能提升用户参与度和使用乐趣。

**Independent Test**: 可以在首页搜索城市后，验证食材展示效果、选择交互和动画反馈。

**Acceptance Scenarios**:

1. **Given** 食材列表已加载, **When** 用户浏览食材, **Then** 看到色彩丰富的食材展示，每个分类有独特的视觉标识
2. **Given** 用户点击选择食材, **When** 选中食材时, **Then** 有愉悦的选中动画效果
3. **Given** 用户想查看食材详情, **When** 点击详情入口, **Then** 弹窗以有趣的动画形式展开
4. **Given** 用户选择了多种食材, **When** 查看已选列表, **Then** 已选食材有清晰且有设计感的展示

---

### User Story 3 - 沉浸式菜谱列表浏览 (Priority: P1)

用户进入菜谱列表页面后，看到的是全屏联动的菜谱展示。每个菜谱占据较大的视觉空间，配图突出，滑动浏览时有流畅的过渡动画。页面整体给人杂志般的阅读体验，而非传统的列表罗列。

**Why this priority**: 菜谱展示是用户获取价值的关键环节，沉浸式设计能提升菜谱的吸引力。

**Independent Test**: 可以通过带参数访问菜谱列表页面，验证菜谱展示效果和滑动体验。

**Acceptance Scenarios**:

1. **Given** 用户进入菜谱列表页面, **When** 页面加载完成, **Then** 菜谱以大图沉浸式卡片形式展示
2. **Given** 用户滑动浏览菜谱, **When** 滚动页面, **Then** 有流畅的视差效果或动画过渡
3. **Given** 用户悬停在菜谱上, **When** 交互时, **Then** 有明显且有趣的悬停效果
4. **Given** 用户想返回或继续, **When** 进行导航操作, **Then** 页面转场有流畅的动画效果

---

### User Story 4 - 活力菜谱详情页 (Priority: P2)

用户点击菜谱进入详情页后，看到的是一个视觉丰富、排版活泼的菜谱展示。配图占据突出位置，食材和步骤以有设计感的方式呈现，整体像一份精心设计的食谱卡片而非枯燥的说明文档。

**Why this priority**: 详情页是用户最终获取价值的页面，活泼的设计能提升用户满意度和分享意愿。

**Independent Test**: 可以通过直接访问菜谱详情页面，验证内容展示效果和视觉吸引力。

**Acceptance Scenarios**:

1. **Given** 用户进入菜谱详情页, **When** 页面加载完成, **Then** 菜谱以杂志风格呈现，配图突出
2. **Given** 用户查看食材清单, **When** 浏览食材区域, **Then** 食材以有设计感的方式列出（如彩色标签、图标）
3. **Given** 用户查看烹饪步骤, **When** 浏览步骤区域, **Then** 步骤编号有视觉设计感，内容易读
4. **Given** 用户想分享或导出, **When** 查看分享/导出功能, **Then** 按钮设计有趣且位置合理

---

### User Story 5 - 食材详情弹窗升级 (Priority: P2)

用户查看食材详情时，弹窗以活泼的动画形式出现，内容展示丰富有趣。可能包含该食材的插画或图标，信息以卡片式分区呈现，关闭时也有流畅的动画效果。

**Why this priority**: 食材详情是补充功能，但精致的弹窗设计能提升整体产品质感。

**Independent Test**: 可以在首页点击食材详情按钮，验证弹窗展示效果和交互体验。

**Acceptance Scenarios**:

1. **Given** 用户点击食材详情, **When** 弹窗打开, **Then** 以有趣的动画形式展开（如弹出、滑入）
2. **Given** 弹窗已打开, **When** 用户浏览内容, **Then** 信息以卡片式分区呈现，有图标或插画装饰
3. **Given** 用户关闭弹窗, **When** 点击关闭, **Then** 弹窗以流畅动画消失

---

### User Story 6 - 全局导航与品牌升级 (Priority: P2)

应用有统一的导航体验，品牌形象升级为更年轻活泼的视觉风格。导航可能采用底部导航栏或侧边抽屉等现代模式，logo和品牌元素更具设计感。

**Why this priority**: 全局导航和品牌形象影响整体产品感知，但不阻塞核心功能使用。

**Independent Test**: 可以在不同页面验证导航一致性和品牌元素展示。

**Acceptance Scenarios**:

1. **Given** 用户在任意页面, **When** 查看导航元素, **Then** 导航设计年轻化，交互有动效
2. **Given** 用户查看品牌元素, **When** 观察 logo 和品牌色, **Then** 视觉风格统一且有活力

---

### User Story 7 - 响应式适配与移动端优化 (Priority: P3)

应用在不同设备上都能呈现良好的视觉效果，移动端有针对性的布局优化，触摸交互友好，沉浸式体验在小屏幕上同样流畅。

**Why this priority**: 响应式设计是锦上添花，核心功能在主流设备上可用即可。

**Independent Test**: 可以在不同屏幕尺寸下验证页面布局和交互效果。

**Acceptance Scenarios**:

1. **Given** 用户在移动设备访问, **When** 页面加载, **Then** 布局适配小屏幕，内容完整可用
2. **Given** 用户在移动端操作, **When** 进行触摸交互, **Then** 触摸反馈友好，动画流畅

---

### Edge Cases

- 低性能设备或动画敏感用户：遵循系统 `prefers-reduced-motion` 设置，自动关闭或简化动画效果
- 深色模式下的色彩适配（如用户系统偏好深色模式）
- 长列表（大量食材或菜谱）的滚动性能优化

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 系统 MUST 呈现活泼年轻的整体视觉风格，包括明亮色彩和有趣的设计元素
- **FR-002**: 首页 MUST 采用全屏沉浸式布局，搜索区域为视觉焦点
- **FR-003**: 食材展示 MUST 采用色彩丰富的视觉设计，每个分类有独特标识
- **FR-004**: 食材选择 MUST 有愉悦的动画反馈效果
- **FR-005**: 菜谱列表 MUST 采用大图沉浸式展示方式，而非传统列表
- **FR-006**: 菜谱详情 MUST 以杂志风格呈现，配图突出，排版活泼
- **FR-007**: 食材详情弹窗 MUST 有流畅的开关动画效果
- **FR-008**: 页面转场 MUST 有流畅的动画过渡效果
- **FR-009**: 系统 MUST 保持所有现有功能的完整性
- **FR-010**: 交互元素 MUST 有生动的悬停和点击反馈
- **FR-011**: 系统 MUST 在主流设备上呈现良好的响应式布局

### Key Entities

- **页面**: 首页（沉浸式搜索）、菜谱列表页（沉浸式浏览）、菜谱详情页（杂志风格）、设置页
- **组件**: 沉浸式搜索框、活力食材卡片、大图菜谱卡片、动效弹窗、活力导航栏
- **视觉元素**: 品牌色彩系统（明亮活泼）、动画效果库、图标/插画元素、季节性背景

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 应用整体视觉风格从"稳重自然"转变为"活泼年轻"，目标用户群体（18-35岁）认可度提升
- **SC-002**: 用户完成核心流程的操作体验流畅，页面转场动画无明显卡顿
- **SC-003**: 首页呈现沉浸式全屏设计，视觉冲击力相比当前版本显著提升
- **SC-004**: 菜谱列表采用大图展示，每个菜谱的视觉吸引力相比列表式提升
- **SC-005**: 所有交互元素有明显且有趣的动效反馈
- **SC-006**: 页面在 1920x1080 桌面屏幕和 375x667 移动屏幕上都能正常展示
- **SC-007**: 页面首次加载时间保持在 3 秒以内，动画不影响核心功能可用性

## Assumptions

- 保留现有的页面路由结构，不改变 URL 设计
- 保留现有的数据流和后端接口，仅改变前端 UI 展现
- 继续使用 Ant Design 作为基础组件库，但会大幅定制样式
- 色彩方案将从鼠尾草绿系转变为珊瑚橙粉系（温暖、年轻、与食物主题关联）
- 动画效果优先保证流畅性，遵循系统 `prefers-reduced-motion` 设置自动降级
- 移动端布局为响应式适配，而非独立的移动端设计

## Out of Scope

- 后端 API 变更
- 新增页面或功能
- 独立的移动端 App 设计
- 国际化支持
- 无障碍访问全面优化（可作为后续功能）
- 深色模式支持（可作为后续功能）
